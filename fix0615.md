# API 模式修复方案

> 目标：将 API 模式从"几乎不可用"修复为可以独立完成编码任务的 Agent。

## 问题诊断

| # | 问题 | 严重度 | 原因 |
|---|------|--------|------|
| 1 | 没有对话历史 | 致命 | `streamText` 只传 `prompt`（单条文本），不传 `messages` 数组，每轮对话都是无记忆的全新会话 |
| 2 | 没有默认 system prompt | 致命 | 模型不知道自己是 Agent、不知道工具行为规范，只靠 `appendSystemPrompt` 传可选自定义 |
| 3 | 权限守卫阻塞工具执行 | 高 | `default` 模式下每个 bash/file_edit/file_write 都要等用户点击，超时 5 分钟 |
| 4 | 没有 `message` 事件 | 高 | `streamText` 只发 `message-delta`，不发完整 `message`，transcript 显示不完整 |
| 5 | 没有 thinking/reasoning 事件 | 中 | 模型的推理过程完全不透传 |
| 6 | 没有 cost 追踪 | 中 | `usage` 事件缺少 `costUsd` |
| 7 | temperature/topP 硬编码 | 低 | 固定 `temperature: 1, topP: 0.95`，不可配置 |
| 8 | maxSteps 硬编码 | 低 | UI 层写死 `apiMaxSteps: 10`，无法调整 |
| 9 | `addDirs` 不支持 | 低 | 参数被忽略 |

---

## 修复方案

### P0：传入对话历史（解决失忆问题）

**思路**：将 `SingleSession.conversation.events` 转为 AI SDK 的 `ModelMessage[]` 传给 `streamText`。

private getConversationHistoryForApi(session: SingleSession): AgentEvent[] {

#### 5. `apiAdapter.ts` 核心改造 — 构建 messages

```typescript
// src/main/adapters/apiAdapter.ts

import type { ModelMessage } from 'ai'

private buildMessages(input: RunTurnInput): ModelMessage[] {
  const messages: ModelMessage[] = []
  
  if (!input.conversationHistory?.length) {
    // 首轮：单条 user message
    messages.push({ role: 'user', content: input.prompt })
    return messages
  }

  // 多轮：从 AgentEvent[] 重建 messages
  let pendingToolCalls: { id: string; name: string; args: unknown }[] = []
  
  for (const ev of input.conversationHistory) {
    switch (ev.kind) {
      case 'system':
        if (ev.text.startsWith('↳ ')) {
          // Flush any pending tool calls before user message
          if (pendingToolCalls.length > 0) {
            messages.push({
              role: 'assistant',
              content: pendingToolCalls.map(tc => ({
                type: 'tool-call' as const,
                toolCallId: tc.id,
                toolName: tc.name,
                args: tc.args
              }))
            })
            pendingToolCalls = []
          }
          messages.push({ role: 'user', content: ev.text.slice(2) })
        }
        break
      case 'message':
        // Flush pending tool calls first
        if (pendingToolCalls.length > 0) {
          messages.push({
            role: 'assistant',
            content: pendingToolCalls.map(tc => ({
              type: 'tool-call' as const,
              toolCallId: tc.id,
              toolName: tc.name,
              args: tc.args
            }))
          })
          pendingToolCalls = []
        }
        messages.push({ role: 'assistant', content: ev.text })
        break
      case 'tool-call':
        pendingToolCalls.push({ id: ev.id, name: ev.name, args: ev.input })
        break
      case 'tool-result':
        // If we have pending tool calls, flush them
        if (pendingToolCalls.length > 0) {
          messages.push({
            role: 'assistant',
            content: pendingToolCalls.map(tc => ({
              type: 'tool-call' as const,
              toolCallId: tc.id,
              toolName: tc.name,
              args: tc.args
            }))
          })
          pendingToolCalls = []
        }
        messages.push({
          role: 'tool',
          content: [{
            type: 'tool-result' as const,
            toolCallId: ev.id,
            result: ev.output
          }]
        })
        break
      // skip: message-delta, thinking, usage, turn-done, error, session-started, file-changed, stderr
    }
  }

  // Flush remaining tool calls
  if (pendingToolCalls.length > 0) {
    messages.push({
      role: 'assistant',
      content: pendingToolCalls.map(tc => ({
        type: 'tool-call' as const,
        toolCallId: tc.id,
        toolName: tc.name,
        args: tc.args
      }))
    })
  }

  // 追加当前用户消息
  messages.push({ role: 'user', content: input.prompt })
  return messages
}
```

### P0：默认 System Prompt

新建 `src/main/adapters/api-tools/systemPrompt.ts`：

```typescript
export function buildSystemPrompt(input: RunTurnInput): string {
  const sections: string[] = []
  
  // 基础身份
  sections.push(`You are a coding agent. You help users with software engineering tasks in their local codebase.`)
  
  // 工具使用指引
  sections.push(`
## Tools
You have access to these tools:
- bash: Execute shell commands. Use for git, npm, running tests, etc.
- file_read: Read file contents with line numbers.
- file_edit: Replace specific text in a file (search & replace).
- file_write: Create or overwrite a file entirely.
- glob: Find files matching a pattern.
- grep: Search file contents with regex.
- fetch: Make HTTP requests.

## Rules
- Read a file before editing it.
- For file_edit: old_string must be an exact, unique match in the file.
- Prefer file_edit over file_write for modifying existing files.
- Use grep/glob to explore the codebase before making assumptions.
- Keep changes minimal — only modify what the task requires.
- Do not add unnecessary comments, error handling, or abstractions.
- When a task is complete, briefly state what you did.`)

  // 工作目录
  sections.push(`\n## Environment\n- Working directory: ${input.cwd}`)
  
  // addDirs
  if (input.addDirs?.length) {
    sections.push(`- Additional directories: ${input.addDirs.join(', ')}`)
  }

  // 用户自定义 system prompt 追加
  if (input.appendSystemPrompt) {
    sections.push(`\n## Additional Instructions\n${input.appendSystemPrompt}`)
  }

  return sections.join('\n')
}
```

在 `apiAdapter.ts` 中把 `system: input.appendSystemPrompt` 替换为 `system: buildSystemPrompt(input)`。

### P1：补全 `message` 事件

在 `mapStreamPart` 中，当收到 `step-finish` 时，如果该 step 有文本输出，发一个完整 `message` 事件。

**更简洁的做法**：在 adapter run loop 中，积累 `text-delta` 的文本，在 `step-finish` 或 `finish` 时发出一个合并的 `message` 事件。

```typescript
// apiAdapter.ts — run() 方法内增加文本积累逻辑
private async run(input: RunTurnInput, queue: AsyncQueue<AgentEvent>): Promise<void> {
  // ...
  let accumulatedText = ''
  
  for await (const part of result.fullStream) {
    if (part.type === 'text-delta') {
      const text = part.textDelta ?? ''
      if (text) {
        accumulatedText += text
        queue.push({ kind: 'message-delta', text })
      }
    }
    if (part.type === 'step-finish' || part.type === 'finish') {
      if (accumulatedText.trim()) {
        queue.push({ kind: 'message', role: 'assistant', text: accumulatedText.trim() })
        accumulatedText = ''
      }
    }
    // ...existing mapping...
  }
}
```

### P1：Cost 追踪

在 `step-finish` 处理中根据 model 和 token 数量估算 cost：

```typescript
// apiAdapter.ts mapStreamPart() step-finish 处理:
case 'step-finish': {
  const usage = isRecord(part.usage) ? part.usage : part
  const inputTokens = numberValue(usage.inputTokens ?? usage.promptTokens)
  const outputTokens = numberValue(usage.outputTokens ?? usage.completionTokens)
  if (inputTokens > 0 || outputTokens > 0) {
    queue.push({
      kind: 'usage',
      inputTokens,
      outputTokens,
      costUsd: estimateCost(modelId, inputTokens, outputTokens)
    })
  }
  return { meaningful: true, turnDone: false, noOutput: false }
}
```

新增 `estimateCost` 函数（简单的价格表查找，先支持主流模型）：

```typescript
const PRICING: Record<string, [number, number]> = {
  // [inputPerMillion, outputPerMillion]
  'claude-sonnet-4-20250514': [3, 15],
  'claude-opus-4-20250514': [15, 75],
  'claude-haiku-4-20250414': [0.8, 4],
  'deepseek-chat': [0.14, 0.28],
  'deepseek-reasoner': [0.55, 2.19],
  'gpt-4o': [2.5, 10],
  'gpt-4o-mini': [0.15, 0.6],
}

function estimateCost(modelId: string, inputTokens: number, outputTokens: number): number {
  // 尝试前缀匹配
  const key = Object.keys(PRICING).find(k => modelId.startsWith(k) || modelId.includes(k))
  if (!key) return 0
  const [inputRate, outputRate] = PRICING[key]
  return (inputTokens * inputRate + outputTokens * outputRate) / 1_000_000
}
```

### P2：`addDirs` 支持

在 `buildSystemPrompt()` 中已经列出 addDirs 信息。同时需要让 `glob`/`grep`/`file_read` 工具能访问这些额外目录：

```typescript
// api-tools/index.ts
export function buildToolSet(
  cwd: string,
  signal: AbortSignal,
  guard: PermissionGuard,
  onFileChanged?: FileChangedCallback,
  addDirs?: string[]  // ← 新增
) {
  const allowedPaths = [cwd, ...(addDirs ?? [])]
  return {
    bash: createBashTool(cwd, signal, guard),
    file_read: createFileReadTool(cwd, allowedPaths),
    file_edit: createFileEditTool(cwd, guard, onFileChanged, allowedPaths),
    file_write: createFileWriteTool(cwd, guard, onFileChanged, allowedPaths),
    glob: createGlobTool(cwd, allowedPaths),
    grep: createGrepTool(cwd, allowedPaths),
    fetch: createFetchTool(),
    sourcegraph: createSourcegraphTool(),
    todo_write: createTodoWriteTool()
  }
}
```

各工具的路径解析函数支持 `allowedPaths` 校验，防止访问非授权目录。

## 验证方式

1. **多轮对话测试**：
   - 选 API 模式，配置一个 provider（DeepSeek 或 Anthropic）
   - 发消息："帮我看一下这个项目的 package.json 里有什么依赖"
   - 等 agent 读完文件返回后，追问："其中哪些是 dev 依赖？"
   - 验证 agent 能回答（证明有历史）

2. **工具调用测试**：
   - 发消息："在当前目录创建一个 hello.txt 写入 hello world"
   - 验证 agent 调了 file_write 并成功
   - 验证不需要点权限确认按钮（bypassPermissions 生效）

3. **流式输出测试**：
   - 验证 transcript 里能看到实时的文字输出
   - 验证最终有完整的 `message` 气泡

4. **Workflow 集成测试**：
   - 创建一个 API vendor 的 agent
   - 放入 workflow 某步
   - 验证能自动跑完并产出 handoff

5. **单元测试**：
   - `tests/api-tools.test.mjs` 已有工具测试，补充 history 转换的单元测试
