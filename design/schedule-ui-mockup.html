总结

方案 A（Tab 切换）：最干净，Runs 和 Schedules 完全分开。Schedule 详情页可以展示历史触发记录。但切 tab 后看不到 runs。

方案 B（折叠面板）：Schedules 和 Runs 在同一视图，一览无余。但左侧栏会比较拥挤。

方案 C（Drawer）：最不侵入现有布局，Schedules 按需打开。但右侧 drawer 会占用 run 详情空间。
