Interactive Mode — UI 增量变更预览

仅展示在现有 agent-studio UI 上的新增/变更部分。所有样式直接复用项目 CSS 变量。

1. Canvas 属性面板 — 新增配置项 NEW

选中步骤节点时，右侧 NodePropertyPanel 底部新增两个配置区。位于现有「错误处理」和「Handoff 解析失败」之后。

Properties
Agent
错误处理
交互模式 NEW
允许步骤内对话
开启后，Agent 未输出 handoff 时步骤暂停等待用户回复
失败策略 NEW
最大重试次数
跳转目标
StepRule 优先于此配置
新增位置：
在 NodePropertyPanel 现有的「错误处理」「Handoff 解析失败」区域下方

交互方式：
• Toggle 开关复用 settings-switch 组件
• 下拉/数字输入复用现有 field + select/input 样式
• 「跳转目标」下拉只在选择「重试后跳转」时出现

数据存储：
写入 node.data 的 interactive 和 failureStrategy 字段，通过 canvasSerializer 序列化到 WorkflowTemplateStep
2. 运行视图 — awaiting-input 状态 NEW

Interactive 步骤的 Agent 输出问题后等待用户回复时。对比：现有的 awaiting-confirm 是黄色，这里用蓝色区分。

与现有 UI 的差异：

1. 状态文字：蓝色「等待回复」（非黄色「等待确认」）

2. Step chip：蓝色边框 + 蓝色脉冲（新增 .workflow-step-chip-awaiting-input CSS 类）

3. 提示条：「Agent 正在等待你的回复」（新增一行 div）

4. 操作按钮：「结束对话，进入下一步」替代「确认并继续」

5. Composer placeholder：「回复 Agent...」替代「向运行中的 agent 发送消息...」

不变的部分：
Transcript 区域、ComposerBar 组件、Rerun/Stop 按钮、右侧文件预览面板 — 全部复用现有组件

4. 运行视图 — 对话结束后自动推进 行为变更

PM Agent 输出 handoff JSON 后，步骤自动完成并推进到下一步。UI 上无需任何人工操作。

行为变更：

Interactive 步骤的 Agent 输出 handoff JSON 后，跳过「确认并继续」环节，直接推进到下一步。

等同于 autoConfirm = true 的行为，但仅对 interactive 步骤生效。

理由：
用户已经在对话过程中跟 Agent 充分沟通过了，不需要再看一遍 handoff 确认。
