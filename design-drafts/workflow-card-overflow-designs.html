方案对比总结

6 种方案均遵循「头部模型标签 + 主体 Agent pills」的布局原则。

方案 A · 智能截断
  • 头部：3 个模型标签（去前缀）+ 步骤计数
  • 主体：完整 Agent pills，渐变遮罩 + "+N" 展开
  • 适用：通用场景，建议作为默认方案
方案 B · 内部滚动
  • 头部：3 个模型标签 + 步骤计数
  • 主体：完整 Agent pills，内部垂直滚动
  • 适用：节点极多（>20）且需频繁查看名称
方案 C · 状态统计
  • 头部：3 个模型标签 + 步骤计数
  • 主体：状态统计徽章 + 当前 Agent mini pill
  • 适用：Dashboard 概览页
方案 D · 节点时间线
  • 头部：3 个模型标签 + 步骤计数
  • 主体：彩色圆点时间线 + 当前 Agent 标签
  • 适用：强调视觉设计感
方案 E · 混合精简
  • 头部：3 个模型标签 + 步骤计数
  • 主体：最近 2-3 个 Agent pills + 统计徽章
  • 适用：兼顾关键信息与总览
方案 F · 紧凑网格
  • 头部：3 个模型标签 + 步骤计数
  • 主体：等宽网格排列 Agent 名字缩写
  • 适用：节点名短且长度相近
