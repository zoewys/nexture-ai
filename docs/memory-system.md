# Agent Memory System V1

Agent Studio 的记忆系统让每个 agent 在一次次的 workflow 运行中积累经验，越用越懂你的项目和偏好。

## 它解决什么问题

没有记忆系统时，每次启动 workflow 都是"从零开始"——agent 不记得你上次确认过什么做法、纠正过什么错误、偏好什么风格。记忆系统在背后自动捕捉这些信号，在下一次运行时把相关经验注入 agent 的上下文，让 agent 少犯重复错误、更快对齐你的预期。

## 它是如何工作的

整个系统由四个组件组成一个闭环：

```
用户操作 → 信号收集 → 反思提取 → 记忆存储 → 下次运行自动注入
   ↑                                                      │
   └──────────────────── 循环 ─────────────────────────────┘
```

### 1. 信号收集 — 只在明确的时刻学习

系统不会不停地"偷听"。只在四种明确信号出现时才触发学习：

| 信号 | 触发时机 | 学到什么 |
|------|---------|---------|
| **用户确认** (positive) | 你在交接点点了"确认并继续" | 这次做得对，记住方法 |
| **用户重跑** (negative) | 你点了"重新运行"并输入了修正指令 | 哪里不够好，下次怎么改 |
| **交接失败** (format-error) | agent 输出的 JSON 无法解析 | 输出格式问题 |
| **流程完成** (completion) | 整个 workflow 顺利跑完 | 全流程中的最佳实践 |

以下场景**不会**触发学习：
- agent 正在运行中
- 你没有在交接点做任何操作
- 单个 step 的运行和停止（只跑不确认 = 不学）

### 2. 反思引擎 — 从 transcript 中提炼经验

信号触发后，系统会用你指定的轻量模型（默认 Claude Haiku）阅读该 step 的运行 transcript，提取出值得记住的**具体、可操作**的经验。垃圾话和笼统建议会被自动过滤掉（confidence 低于 0.6 的经验直接丢弃）。

每条记忆带有一个分类标签：

| 分类 | 含义 | 示例 |
|------|------|------|
| → 方法 | 如何做某事 | "先列出边界条件再写实现" |
| 📌 知识 | 项目/技术事实 | "这个项目使用 React 18 + TypeScript" |
| ✓ 偏好 | 你喜欢什么 | "用户偏好短列表，不要超过 5 条" |
| ⚠️ 避免 | 不要做什么 | "不要在这个项目中使用 any 类型" |

### 3. 记忆存储 — JSON 文件，两层作用域

所有记忆以 JSON 文件存在本地，不依赖任何外部数据库：

```
<用户数据目录>/memories/
├── config.json           # 反思引擎全局配置
├── agents/
│   └── <agentId>/
│       ├── meta.json         # 该 agent 的统计信息
│       ├── global.json       # 跨项目通用的记忆
│       └── projects/
│           └── <hash>.json   # 特定项目的记忆
└── raw/                     # 暂存的未处理信号
```

每一条记忆有两层作用域：
- **全局 (global)**：跨所有项目适用，如通用编程方法论
- **项目 (project)**：仅适用于特定项目，如"这个项目 React 版本是 18"

### 4. 运行时注入 — 在 prompt 中植入经验

当你启动一个 workflow 时，系统会自动检索该 agent 的相关记忆（同项目 + 全局），按重要程度排序后注入到 agent 的 system prompt 前面。agent 会看到类似这样的上下文：

```
# 你的积累经验

⚠️ 避免：不要在这个项目中使用 any 类型
→ 方法：先列出边界条件再写实现
✓ 偏好：用户偏好短列表，不要超过 5 条

请参考以上经验，但根据当前具体情况灵活应用。
```

注入的记忆数量受到 token 预算限制（默认约 1500 token），记忆太多时只保留最重要的几条。

## 遗忘曲线 — 用进废退

每条记忆有一个强度值 (0–1)，会随时间自然衰减。但每次被使用且 run 成功后会**强化**：强度回到 1，衰减速度变慢。

- 新记忆衰减最快（7 天半衰期）
- 被强化过的记忆衰减更慢
- 强度低于阈值 (0.3) 的记忆会被自动过滤，不再注入

这意味着：频繁用到的经验会沉淀下来，偶尔一次的经验会自然消失。

## 在界面中使用

### 查看 Agent 的记忆

1. 打开 **Agents** 页面
2. 选中一个 agent，进入编辑模式
3. 展开底部的 **记忆** 面板

面板会显示：
- 记忆总数统计
- **全局** tab：跨项目通用的记忆
- **项目: xxx** tab：每个使用过该 agent 的项目的专属记忆
- 每条记忆的分类标签、强度、强化次数、创建时间
- **删除**按钮：移除不需要的记忆

### 配置反思引擎

1. 在 **Agents** 编辑页面中，找到 **记忆 / 反思设置** 面板
2. 可以调整：
   - **启用/关闭**：关闭后，不再产生新记忆（已有记忆仍会注入）
   - **Vendor**：选择用于反思的 CLI（claude / codex）
   - **Model**：选择具体的模型（默认 Claude Haiku，轻量够用）

> 💡 建议使用轻量模型（如 Haiku）来做反思，反思只需要简单的文本分析，不需要大模型。

## 当前限制 (V1)

V1 是一个**最小可用闭环**，以下能力不在当前版本中：

- ❌ **Single Run 模式**：只对 Workflow Run 生效，单 agent 模式下不产生记忆
- ❌ **编辑记忆**：只能查看和删除，不能手动创建或修改
- ❌ **手工添加记忆**：记忆只能由反思引擎自动生成
- ❌ **向量检索**：基于分类和强度的简单排序，不做语义检索
- ❌ **跨设备同步**：记忆存储在本地文件中，不自动同步

后续版本计划：
- Skill Library：从重复出现的记忆中提取可复用的技能模板
- 语义检索：用 embedding 匹配最相关的经验
- 记忆编辑：手动补充和修正经验

## 常见问题

**Q: 记忆会占用多少磁盘空间？**
A: 极少。每条记忆只有几百字节，加上 transcript 摘要通常不超过几 KB。即使数百条记忆也只有几 MB。

**Q: 关闭反思后已有记忆会消失吗？**
A: 不会。关闭只会停止产生新记忆，已有的记忆仍然会被注入 prompt。

**Q: 删除一个 agent 后它的记忆还在吗？**
A: 不会。删除 agent 时会自动清理该 agent 的所有记忆文件。

**Q: 我能在不同项目间共享记忆吗？**
A: 全局记忆（scope=global）会自动跨项目共享；项目记忆（scope=project）只在该项目下出现。系统会根据信号来源自动判断作用域。

**Q: 如果反思模型不可用怎么办？**
A: 信号会暂存在 `raw/` 目录下，下次反思成功时处理。不会丢失。
